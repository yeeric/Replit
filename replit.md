# CISC 332 Conference Management Dashboard

A server-rendered conference management web application built with PHP, HTMX, and Tailwind CSS. No JavaScript framework required — all interactivity is handled by HTMX HTML attributes.

## Architecture

- **Frontend**: PHP-generated HTML pages with HTMX for dynamic updates, Tailwind CSS (CDN) for styling
- **Backend**: PHP 8.2 with PDO + PostgreSQL (all in `php/` directory)
- **Database**: PostgreSQL (Replit-provisioned)
- **Server**: PHP built-in server on port 5000

## Startup

The "Start application" workflow runs `bash start.sh`:
```bash
nix-shell -p php -p php82Extensions.pdo_pgsql --run "php -S 0.0.0.0:5000 php/index.php"
```

## How It Works

1. PHP serves full HTML pages for all 7 routes
2. HTMX intercepts form submissions and link clicks, sending requests in the background
3. PHP detects HTMX requests via the `HX-Request` header and returns HTML fragments instead of full pages
4. HTMX swaps the returned HTML into the target element (no page reload)
5. Modals use the native HTML `<dialog>` element opened with `showModal()`

## PHP File Structure

```
php/
  index.php       - URL router, dispatches to page handlers
  db.php          - PDO connection (reads PG* env vars)
  layout.php      - renderLayout(), sidebar(), pageHead() helpers
  router.php      - Legacy JSON API (kept for reference, not used by HTML frontend)
  pages/
    dashboard.php  - Finance Overview (SVG pie chart generated by PHP)
    committees.php - Committees (HTMX dropdown → members table)
    hotels.php     - Hotel Rooms (HTMX dropdown → students table)
    schedule.php   - Schedule (HTMX tabs + edit session modal)
    sponsors.php   - Sponsors & Companies (HTMX add/delete company)
    jobs.php       - Job Board (HTMX company filter)
    attendees.php  - Attendees (HTMX tabs + add attendee modal)
```

## Features (Pages)

1. **Finance Overview** (`/`) — Stat cards + PHP-generated SVG pie chart + fee structure table
2. **Committees** (`/committees`) — Dropdown select triggers HTMX fetch of members table
3. **Hotel Rooms** (`/hotels`) — Dropdown select triggers HTMX fetch of assigned students
4. **Schedule** (`/schedule`) — Date tabs (HTMX), session cards, edit dialog with HTMX form submission
5. **Sponsors & Companies** (`/sponsors`) — Sponsor list + company management (HTMX add/delete)
6. **Job Board** (`/jobs`) — Job table with HTMX company filter
7. **Attendees** (`/attendees`) — Type tabs (HTMX), add attendee modal with type-specific fields

## Database Schema

Tables (from SQL script):
- `company`, `committeemember`, `subcommittee`, `hotelroom`, `session`
- `attendee`, `student`, `professional`, `sponsor`, `speaker`
- `jobad`, `attends`, `speaksat`, `memberofcommittee`

## Key HTMX Patterns Used

- `hx-get` / `hx-post` / `hx-delete` — HTTP requests on user interaction
- `hx-target` — CSS selector of element to update
- `hx-swap` — how to swap (innerHTML, outerHTML, beforeend)
- `hx-trigger` — event that fires the request (change, submit, etc.)
- `HX-Trigger` response header — PHP sends this to trigger client-side events (e.g., close modal)
- `hx-indicator` — show loading state during requests
- `hx-confirm` — native browser confirm dialog before delete
